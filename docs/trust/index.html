Infrastructure

Infrastructure Security

Multiple layers of defense protect the platform at every level, from network edge to application runtime.

Encryption

  • AES-256 encryption at rest for all stored data
  • TLS 1.3 enforced for all data in transit
  • SHA-256 hash-chained audit ledger
  • Secrets managed via environment injection, never hardcoded

Access Control

  • Role-based access control (RBAC) with scoped API keys
  • JWT + API key dual authentication
  • Multi-factor authentication for admin access
  • Principle of least privilege enforced

Network Security

  • CORS restricted to approved origins only
  • HSTS enforced (2-year max-age, preload)
  • Permissions-Policy (camera/mic/geo disabled)
  • Redis-backed rate limiting per API key tier

Container Security

  • Non-root Docker containers in production
  • Multi-stage builds with minimal attack surface
  • Health checks with automated restart policies
  • Image scanning for known CVEs before deployment
Data Handling

Data Handling & Privacy

We treat your data with the highest level of care. Here is exactly how we store, process, and protect it.

Data Residency

All customer data is stored in Railway-managed infrastructure with PostgreSQL databases. Data processing occurs in secure, isolated environments. Customers may request specific data residency arrangements under enterprise agreements.

Encryption Standards

AES-256 encryption at rest for all database records and backups. TLS 1.3 for all data in transit. Audit ledger entries are additionally protected by SHA-256 hash chains ensuring tamper-proof integrity.

Retention Policies

Audit logs are retained for 7 years (configurable per customer). Operational data retained for the duration of the service agreement plus 90 days. Upon termination, all customer data is permanently deleted within 30 days upon request.

Data Processing Agreement

A comprehensive DPA is available upon request covering GDPR Article 28 requirements, Standard Contractual Clauses (SCCs), and specific technical and organizational measures. Contact security@governlayer.ai to request a copy.

Security Testing

Penetration Testing & Vulnerability Management

Continuous security validation ensures our defenses remain effective against evolving threats.

Continuous Security Validation

GovernLayer conducts regular security assessments including automated SAST/DAST scanning on every deployment, dependency vulnerability monitoring via GitHub Dependabot, and periodic third-party penetration testing. Our AI-specific security testing covers MITRE ATLAS and OWASP ML Top 10 threat vectors through our built-in threat analysis agent.

Security findings are triaged and remediated according to severity: Critical (24h), High (72h), Medium (2 weeks), Low (next sprint). Penetration test reports are available to enterprise customers under NDA.

All customer data is stored in Railway-managed PostgreSQL databases. Railway operates on major cloud providers with data centers in the US and EU. Enterprise customers can negotiate specific data residency requirements. Cached data in Redis is ephemeral and automatically purged based on TTL settings.
We send only governance-related prompts (policy text, compliance frameworks, risk descriptions) to LLM providers. The audit ledger uses a SHA-256 hash chain, similar to blockchain technology. Each record stores the hash of the previous record plus its own computed hash, creating a tamper-evident chain. The genesis hash is derived from a known seed value. Any modification to a historical record would break the chain and be immediately detectable. Records are stored in PostgreSQL with AES-256 encryption at rest.
GovernLayer supports dual authentication: JWT bearer tokens for interactive sessions and scoped API keys (prefixed with gl_) for programmatic access. API keys can be scoped to specific operations (govern, audit, risk, scan) following the principle of least privilege. Admin accounts support multi-factor authentication. Service accounts can be created via the automation API for bot-to-bot communication.
All API endpoints are protected by Redis-backed rate limiting, tiered by subscription plan: Free (20 rpm), Starter (100 rpm), Pro (500 rpm), Enterprise (2,000 rpm). Rate limits are applied per API key with sliding window counters. Exceeding the limit returns a 429 response with a Retry-After header. Enterprise customers can request custom rate limits. DDoS protection is provided at the infrastructure level by Railway.
