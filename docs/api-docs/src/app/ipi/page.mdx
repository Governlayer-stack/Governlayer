# IPI Scanner

Scan your AI agents for Indirect Prompt Injection vulnerabilities before they reach production. Deterministic engine — no LLM calls, instant results. Every finding maps to specific compliance framework clauses. {{ className: 'lead' }}

## What it detects

| Category | Description | Example |
| --- | --- | --- |
| **Instruction Hierarchy** | Missing system prompt defenses | No priority enforcement between system/user/retrieved content |
| **Data Exfiltration** | Tool-call paths that leak data | Unrestricted HTTP tool can send data to attacker endpoints |
| **Content Boundary** | Untrusted data treated as instructions | Retrieved emails/documents executed as agent commands |
| **Tool Abuse** | Excessive or unrestricted tool access | Code execution, database access without sandboxing |
| **Injection Surface** | Active injection payloads in content | "Ignore previous instructions", model delimiters, unicode hiding |
| **Privilege Escalation** | Agent permissions beyond what's needed | Shell access, unrestricted file writes |

## Scan an agent {{ tag: 'POST', label: '/v1/ipi/scan' }}

<Row>
  <Col>

    Analyze an agent's system prompt, tool configuration, and content for IPI vulnerabilities. Returns a resistance score (0-100), findings sorted by severity, and compliance framework mappings.

    ### Required attributes

    <Properties>
      <Property name="system_name" type="string">
        Name of the agent or system to scan.
      </Property>
    </Properties>

    ### Optional attributes

    <Properties>
      <Property name="system_prompt" type="string">
        The agent's system prompt. Analyzed for missing defenses (instruction hierarchy, data boundaries, output restrictions).
      </Property>
      <Property name="tools" type="array">
        Agent's tool definitions. Each object should have at least a `name` field. Include `restrictions` or `allowlist` to show tools are properly constrained.
      </Property>
      <Property name="content" type="string">
        Retrieved or user content to scan for injection payloads before the agent processes it.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="POST" label="/v1/ipi/scan">

    ```bash {{ title: 'cURL' }}
    curl -X POST https://api.governlayer.ai/v1/ipi/scan \
      -H "X-API-Key: gl_your_api_key_here" \
      -H "Content-Type: application/json" \
      -d '{
        "system_name": "customer-support-agent",
        "system_prompt": "You are a helpful customer support agent. Answer questions about our products.",
        "tools": [
          {"name": "search_knowledge_base"},
    ```

## Framework crosswalk {{ tag: 'GET', label: '/ipi/crosswalk' }}

<Row>
  <Col>

    Get the IPI-to-compliance framework control crosswalk. Maps every IPI vulnerability category to specific clauses across 12 frameworks: NIST AI RMF, ISO 42001, EU AI Act, SOC 2, GDPR, HIPAA, PCI DSS, NIST CSF, ISO 27001, CCPA, NIS2, and DORA.

    ### Optional parameters

    <Properties>
      <Property name="category" type="string">
        Filter by IPI category: `injection_surface`, `data_exfiltration`, `instruction_hierarchy`, `tool_abuse`, `content_boundary`, `privilege_escalation`.
      </Property>
      <Property name="framework" type="string">
        Filter by framework code: `SOC2`, `GDPR`, `EU_AI_ACT`, `NIST_AI_RMF`, `ISO42001`, `HIPAA`, `PCI_DSS`, etc.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="GET" label="/ipi/crosswalk?framework=SOC2">
