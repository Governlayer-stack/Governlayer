# Consensus Engine

For high-stakes governance decisions, GovernLayer's consensus engine queries multiple LLMs independently and requires agreement before acting. Three strategies eliminate single-model hallucination risk. {{ className: 'lead' }}

## Strategies

- **Voting** — 3+ models evaluate independently; majority rules
- **Chain-of-Verification** — Generate, question, verify, synthesize across models
- **Adversarial Debate** — One model claims, another critiques, a third judges

---

## Run consensus {{ tag: 'POST', label: '/v1/achonye/consensus' }}

<Row>
  <Col>

    Run a governance task through multi-model consensus.

    ### Required attributes

    <Properties>
      <Property name="task" type="string">
        The governance question or decision to evaluate.
      </Property>
      <Property name="strategy" type="string">
        Consensus strategy: `voting`, `chain_of_verification`, or `adversarial_debate`.
      </Property>
    </Properties>

    ### Optional attributes

    <Properties>
      <Property name="models" type="array">
        Specific models to use. Defaults to the Board (Sonnet, Gemini, GPT-4o).
      </Property>
      <Property name="min_agreement" type="float">
        Minimum agreement threshold for voting (default: 0.67).
      </Property>
    </Properties>

  </Col>
