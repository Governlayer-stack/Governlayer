API Documentation

Integrate AI governance, compliance auditing, drift detection, and risk scoring into your systems with a single API.

Authentication

All API requests require authentication via either a JWT Bearer token (from login) or an API Key (header: X-API-Key: gl_xxx...). API keys are scoped per-organization with fine-grained permissions.

# Bearer token curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \ https://www.governlayer.ai/v1/scan # API Key curl -H "X-API-Key: gl_your_api_key_here" \ https://www.governlayer.ai/v1/scan

Core Endpoints

POST /v1/govern Auth required
Run full governance pipeline: drift detection + risk scoring + compliance decision + audit ledger entry
POST /v1/scan Auth required
Quick deterministic scan — instant risk assessment without LLM calls
POST /v1/risk Auth required
6-dimension risk score: data sensitivity, model risk, compliance gap, operational, reputational, third-party
POST /v1/drift Auth required
Behavioral drift detection using sentence-transformer embeddings vs safety manifolds
POST /v1/audit Auth required
LLM-powered compliance audit against 9 regulatory frameworks
GET /v1/ledger Auth required
Immutable hash-chained audit trail — tamper-proof governance record

Auth & Account

POST /auth/register
Create a new account. Returns JWT token.
POST /auth/login
Authenticate with email/password. Supports MFA.
GET /auth/oauth/{provider}/authorize
Get OAuth authorization URL (google, microsoft, github)
POST /auth/forgot-password
Send password reset email

Enterprise

POST /v1/enterprise/orgs Auth required
Create an organization (multi-tenant isolation)
POST /v1/enterprise/orgs/{slug}/api-keys Auth required
Generate scoped API keys for programmatic access
GET /v1/enterprise/orgs/{slug}/usage Auth required
View API usage stats and rate limit status

curl -X POST https://www.governlayer.ai/v1/govern \ -H "X-API-Key: gl_your_key" \ -H "Content-Type: application/json" \ -d '{ "system_name": "customer-support-bot", "trace": "User asked about refund policy. Bot recommended full refund without manager approval.", "context": "Production chatbot handling financial decisions", "has_pii": true, "has_ai": true, "framework": "SOC2" }'
